## Core ideas

- A single `createDocsSource(...)` call builds your content model, navigation, metadata, route helpers, heading extraction, and search.
- A single `DocsLayout` owns the docs shell while the theme system handles presets, density, sizing, and accent changes.
- When the default header or sidebar is not enough, you can replace either one with your own component or `null`.
- Locale folders like `entry/en.mdx`, `entry/es.mdx`, and `entry/zh.mdx` map directly into locale-aware URLs.

## What this example site shows

- A single `docs` route powered by one layout component
- Theme presets, density, radius, and shell sizing in one theme studio
- Locale-aware routes for English, Spanish, and Chinese
- A component library for docs homes, guides, and references
