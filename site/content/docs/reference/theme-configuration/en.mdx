## Theme prop

```tsx
import { DocsLayout, ThemeSwitcher } from "@emcy/docs";

<DocsLayout
  navigation={docsSource.getNavigation()}
  theme={{
    preset: "neutral",
    mode: "light",
    accentHue: 38,
    layoutWidth: "1500px",
    contentWidth: "48rem",
    sidebarWidth: "276px",
    tocWidth: "240px",
    radius: "xl",
  }}
  themeSwitcher={<ThemeSwitcher />}
/>
```

## What the knobs do

<Accordion>
  <AccordionItem title="preset and mode" defaultOpen>
    `preset` selects the base palette, surfaces, and contrast model. `mode` lets you force light or dark when the preset supports both.
  </AccordionItem>
  <AccordionItem title="accentHue">
    Use a single hue number when you want to keep the same preset but move the brand accent to blue, teal, rose, orange, or any other value.
  </AccordionItem>
  <AccordionItem title="layoutWidth, contentWidth, sidebarWidth, tocWidth">
    These values let one layout component cover wider product docs, denser reference views, and narrower reading-focused pages without splitting the package.
  </AccordionItem>
  <AccordionItem title="radius">
    Choose `md`, `lg`, or `xl` to tighten or soften the overall surface language across cards, banners, tabs, and shell chrome.
  </AccordionItem>
</Accordion>

## Optional switcher slots

<Callout type="tip" title="The shell only provides slots">
  EmcyDocs does not force a specific language picker or theme picker. Pass your own `languageSwitcher` and `themeSwitcher` nodes when you want them, or omit them when the product should stay fixed.
</Callout>

The example site uses one layout API and changes the feel entirely through preset, mode, density, radius, and shell sizing.
