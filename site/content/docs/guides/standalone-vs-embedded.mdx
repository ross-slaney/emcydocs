## Standalone mode

Standalone mode is for a dedicated docs site. The package owns the docs header, search bar, navigation shell, and mobile docs chrome.

## Embedded mode

Embedded mode is for apps that already have a marketing or product shell. In that case the package renders only the docs frame and accepts a host header or outer layout around it.

## Why this matters

The content model should not care whether the docs are standalone or embedded. One package should support both without forcing you to maintain two separate docs systems.
