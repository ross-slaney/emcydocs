EmcyDocs 支持两种部署模式。选择最适合你项目的那一种。

## 独立模式

在 **独立模式** 下，EmcyDocs 提供完整的文档体验：

- 带品牌、搜索和导航链接的 header
- 带章节导航的 sidebar
- 针对移动端优化的抽屉式导航
- 完整页面控制权

这非常适合：

- 独立文档站，例如 `docs.yourapp.com`
- 开源项目文档
- API 参考站点

```tsx
// app/docs/layout.tsx
import { DocsLayout } from "@emcy/docs";

export default function Layout({ children }) {
  return (
    <DocsLayout
      navigation={docsSource.getNavigation()}
      brand={<span>My Docs</span>}
      topLinks={[
        { href: "/api", label: "API" },
        { href: "https://github.com/...", label: "GitHub" },
      ]}
    >
      {children}
    </DocsLayout>
  );
}
```

## 嵌入模式

在 **嵌入模式** 下，EmcyDocs 只渲染文档内容区域，由你的应用提供外层外壳：

- 没有 header，由你的应用负责 header
- 只保留 sidebar 和内容框架
- 与现有导航集成

这非常适合：

- 应用内文档的 SaaS 产品
- 带文档区的营销站点
- 需要把文档和其他产品功能并排展示的应用

```tsx
// app/docs/layout.tsx
import { DocsLayout } from "@emcy/docs";
import { AppShell, AppHeader } from "@/components/app";

export default function Layout({ children }) {
  return (
    <AppShell>
      <AppHeader />
      <DocsLayout
        mode="embedded"
        mobileHeaderId="app-header"
        navigation={docsSource.getNavigation()}
      >
        {children}
      </DocsLayout>
    </AppShell>
  );
}
```

> `mobileHeaderId` 属性会告诉 EmcyDocs 把移动端文档栏放在你的应用 header 下方。

## 试试这两种模式

这个示例站点同时演示了两种模式：

| 模式 | URL | 你会看到什么 |
|------|-----|--------------|
| 独立 | [docs](../../docs) | EmcyDocs 接管整个页面 |
| 嵌入 | [embedded/docs](../../embedded/docs) | 文档渲染在宿主外壳内部 |

内容来自同一组 MDX 文件。变化的只有外层 chrome。

## 如何选择

| 场景 | 推荐 |
|------|------|
| 公共文档站 | 独立 |
| 开发者 API 参考 | 独立 |
| 应用内帮助中心 | 嵌入 |
| 带文档区的营销站 | 嵌入 |
| Dashboard 内的产品文档 | 嵌入 |
