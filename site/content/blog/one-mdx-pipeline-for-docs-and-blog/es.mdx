# Una sola canalizacion MDX para docs y blog

Cuando el blog vive dentro de la libreria, duplicar el renderer MDX deja de tener sentido.

## Render compartido para autoria consistente

Las personas autoras pueden seguir usando los mismos callouts, cards, tabs y bloques de codigo en docs y en el blog.

## Una sintaxis

Reutilizar el mismo mapa de componentes significa que la libreria solo mantiene un contrato MDX.

## Un tema

Las variables del tema de docs ya definen superficies, bordes y color. El blog debe consumir esos mismos tokens.
