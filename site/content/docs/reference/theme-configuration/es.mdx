## Theme prop fijo

```tsx
const docsTheme = {
  color: {
    preset: "ocean",
    mode: "dark",
    accentHue: 188,
    accentStrength: "bold",
    surfaceStyle: "elevated",
  },
  layout: {
    density: "compact",
    layoutWidth: "1480px",
    contentWidth: "50rem",
    sidebarWidth: "272px",
    tocWidth: "232px",
  },
  shape: {
    radius: "xl",
  },
};

<DocsLayout
  navigation={docsSource.getNavigation()}
  theme={docsTheme}
/>
```

<Callout type="tip" title="Tema fijo o tema vivo">
  `DocsLayout` lee primero el estado de `DocsThemeProvider` cuando existe y, si no, usa su prop `theme`. Usa la prop para sitios fijos y el provider para un studio o preferencias del usuario.
</Callout>

## Estado de tema en vivo

```tsx
import { DocsThemeProvider, useDocsTheme } from "@emcy/docs";

function ThemeModeToggle() {
  const { resolvedTheme, updateTheme } = useDocsTheme();
  const nextMode =
    resolvedTheme.config.color.mode === "light" ? "dark" : "light";

  return (
    <button onClick={() => updateTheme({ color: { mode: nextMode } })}>
      Cambiar modo
    </button>
  );
}
```

## Que hace cada opcion

<Accordion>
  <AccordionItem title="color.preset y color.mode" defaultOpen>
    `preset` define la familia base de color. `mode` fuerza light o dark, y todos los presets soportan ambos.
  </AccordionItem>
  <AccordionItem title="color.accentHue y color.accentStrength">
    `accentHue` mueve el color de marca sin abandonar el preset. `accentStrength` decide cuanto peso tiene ese acento en botones, focus rings y superficies suaves.
  </AccordionItem>
  <AccordionItem title="color.surfaceStyle">
    `flat`, `tinted` y `elevated` cambian cuanto tinte, contraste de borde y profundidad de sombra usa el shell.
  </AccordionItem>
  <AccordionItem title="layout.density y anchos">
    `density` cambia el ritmo de espaciado en nav, cards, TOC y gutters. Los anchos cubren shells mas amplios, referencias mas densas y vistas mas centradas en lectura.
  </AccordionItem>
  <AccordionItem title="shape.radius">
    `md`, `lg` y `xl` endurecen o suavizan cards, banners, tabs, dialogs y chrome.
  </AccordionItem>
  <AccordionItem title="tokens">
    Los `tokens` son overrides semanticos que se aplican al final. Usalos cuando necesites igualar valores exactos de fondo, borde, card, acento, codigo o sombras.
  </AccordionItem>
</Accordion>

## Studio del sitio vs API de libreria

<Callout type="tip" title="El popup de este sitio vive solo en el ejemplo">
  `DocsThemeStudio` solo existe en `/site`. La libreria publica `resolveDocsTheme(theme)`, `DocsThemeProvider` y `useDocsTheme()`. Puedes construir tu propio switcher o no tener controles en runtime.
</Callout>

El sitio de ejemplo usa esas primitivas para presets rapidos, overrides de tokens, URLs compartibles y preferencias persistidas sin depender de un switcher empaquetado por la libreria.
