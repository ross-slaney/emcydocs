## Por que importa

Es facil subestimar los docs mobile. La busqueda, la navegacion, los controles sticky y el bloqueo de scroll se vuelven incomodos muy rapido cuando se reconstruyen de forma ad hoc en cada proyecto.

## Lo que incluye EmcyDocs

- una barra mobile para docs
- un drawer de navegacion mobile
- un punto de entrada de busqueda mobile
- bloqueo del scroll del body cuando el drawer esta abierto
- offsets conscientes del header para docs embebidos

## Resultado

La UX por defecto cubre las partes molestas para que los equipos de producto puedan concentrarse en el contenido y el branding en vez de rehacer el chrome mobile desde cero.
