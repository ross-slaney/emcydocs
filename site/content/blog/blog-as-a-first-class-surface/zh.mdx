# 博客成为一等内容界面

如果每个新项目都要重新搭建博客，说明这个能力本来就应该进入库本身。

## 为什么要和文档并列？

文档系统已经解决了几个关键问题：

- 内容直接放在代码仓库里
- 按语言感知的路由
- 适合预渲染的 metadata
- 一套共享的 MDX 渲染流程

博客应该复用这些决定，而不是再建立第二套几乎相同的内容系统。

<Callout type="info" title="共享基础设施很重要">
  目标不是把博客文章伪装成文档页面，而是让两种内容界面建立在同一套可靠的内容与渲染基础之上。
</Callout>

## 对使用者来说意味着什么？

现在可以直接获得博客内容源、目录页、详情页，以及分类、搜索和推荐文章这些常见能力。

## 什么仍然由应用决定？

外层营销壳、页面文案以及额外品牌元素仍然由使用该库的应用自行决定。
