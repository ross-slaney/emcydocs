## Theme prop

```tsx
<DocsLayout
  navigation={docsSource.getNavigation()}
  theme={{
    preset: "neutral",
    mode: "light",
    accentHue: 38,
    layoutWidth: "1500px",
    contentWidth: "48rem",
    sidebarWidth: "276px",
    tocWidth: "240px",
    radius: "xl",
  }}
  themeSwitcher={<ThemeSwitcher />}
/>
```

## Que hace cada opcion

<Accordion>
  <AccordionItem title="preset y mode" defaultOpen>
    `preset` define la paleta y las superficies. `mode` fuerza light u dark cuando ese preset lo soporte.
  </AccordionItem>
  <AccordionItem title="accentHue">
    Cambia solo el color de acento sin abandonar el preset base.
  </AccordionItem>
  <AccordionItem title="layoutWidth, contentWidth, sidebarWidth, tocWidth">
    Permite cubrir docs clasicos, notebook, embedded y minimal con el mismo componente de layout.
  </AccordionItem>
  <AccordionItem title="radius">
    Elige `md`, `lg` o `xl` para endurecer o suavizar el lenguaje visual.
  </AccordionItem>
</Accordion>

## Slots opcionales

<Callout type="tip" title="El shell solo reserva el lugar">
  EmcyDocs no obliga un selector de idioma o tema concreto. Puedes pasar `languageSwitcher` y `themeSwitcher` o dejar ambos vacios.
</Callout>
