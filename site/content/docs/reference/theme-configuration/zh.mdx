## 固定 theme prop

```tsx
const docsTheme = {
  color: {
    preset: "ocean",
    mode: "dark",
    accentHue: 188,
    accentStrength: "bold",
    surfaceStyle: "elevated",
  },
  layout: {
    density: "compact",
    layoutWidth: "1480px",
    contentWidth: "50rem",
    sidebarWidth: "272px",
    tocWidth: "232px",
  },
  shape: {
    radius: "xl",
  },
};

<DocsLayout
  navigation={docsSource.getNavigation()}
  theme={docsTheme}
/>
```

<Callout type="tip" title="固定主题与实时主题">
  `DocsLayout` 会优先读取 `DocsThemeProvider` 中的主题状态；如果没有 provider，才会退回到自己的 `theme` prop。固定站点直接传 prop，需要主题工作室或用户偏好时再加 provider。
</Callout>

## 实时主题状态

```tsx
import { DocsThemeProvider, useDocsTheme } from "@emcy/docs";

function ThemeModeToggle() {
  const { resolvedTheme, updateTheme } = useDocsTheme();
  const nextMode =
    resolvedTheme.config.color.mode === "light" ? "dark" : "light";

  return (
    <button onClick={() => updateTheme({ color: { mode: nextMode } })}>
      切换模式
    </button>
  );
}
```

## 参数说明

<Accordion>
  <AccordionItem title="color.preset 和 color.mode" defaultOpen>
    `preset` 负责基础色系，`mode` 用来强制 light 或 dark。现在所有 preset 都同时支持两种模式。
  </AccordionItem>
  <AccordionItem title="color.accentHue 和 color.accentStrength">
    `accentHue` 用来重新设定品牌强调色的色相，`accentStrength` 决定它对按钮、焦点环和强调表面的影响强度。
  </AccordionItem>
  <AccordionItem title="color.surfaceStyle">
    `flat`、`tinted` 和 `elevated` 会改变面板的染色程度、边框对比度和阴影深度。
  </AccordionItem>
  <AccordionItem title="layout.density 与宽度">
    `density` 会改变导航、cards、TOC 和页面留白的节奏。宽度配置则让同一个布局既能覆盖更宽的产品文档，也能覆盖更紧凑的参考页面。
  </AccordionItem>
  <AccordionItem title="shape.radius">
    通过 `md`、`lg` 或 `xl` 调整 cards、banner、tabs、dialog 和 shell chrome 的圆润程度。
  </AccordionItem>
  <AccordionItem title="tokens">
    `tokens` 是最后应用的语义覆盖层。当前面的 preset 和旋钮已经接近目标，但你还需要精确匹配背景、边框、card、强调色、代码区或阴影时，就用它。
  </AccordionItem>
</Accordion>

## 示例站工作室 vs 库 API

<Callout type="tip" title="这个站点里的弹窗只属于示例应用">
  `DocsThemeStudio` 只存在于 `/site`。库本身只提供 `resolveDocsTheme(theme)`、`DocsThemeProvider` 和 `useDocsTheme()` 这些 headless 原语。你可以自己做切换器，也可以完全不暴露运行时主题控制。
</Callout>

示例站点正是基于这些原语实现了预设切换、token 覆盖、可分享链接和持久化偏好，而不是依赖一个内置的库级 switcher 组件。
